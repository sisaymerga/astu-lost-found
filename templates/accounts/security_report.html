Security Report

Vulnerability: Insecure Direct Object References (IDOR)

How it could be exploited: An attacker could manipulate the item ID in the claim URL (e.g., /claim/5/) to claim an item they do not own or that is not found, by simply changing the ID in the browser.

How it was prevented: In the submit_claim view, we filter the item by id=item_id, status='found'. This ensures that only items with status 'found' can be claimed. Additionally, we use @login_required to ensure only authenticated users can access the claim view. For extra security, we also check that the item hasn't already been claimed and that the user hasn't already submitted a claim for the same item.

Another protection: We never expose internal IDs in a way that allows privilege escalation, and we always verify ownership/permissions on the server side.
